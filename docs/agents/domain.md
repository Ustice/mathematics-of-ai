# Domain Documentation

This is a single-context repository for one living Mathematics of AI course and its publishing application.

## Consumer Rules

Before architecture, diagnosis, or test-design work:

1. Read the root `CONTEXT.md` when it exists.
2. Read relevant architectural decisions under `docs/adr/` when that directory exists.
3. Use the course vocabulary defined in `CONTEXT.md` rather than inventing synonyms.
4. Surface any proposal that conflicts with an ADR instead of silently overriding it.

`CONTEXT.md` and ADRs are created lazily as domain terms and durable decisions become concrete. Their absence does not block ordinary repository work.
