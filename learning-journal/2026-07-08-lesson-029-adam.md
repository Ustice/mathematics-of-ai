# Learning Journal Entry - 2026-07-08

## Lesson / Topic

Lesson 29 - Adam.

## What Clicked

Adam can be taught as two optimizer memories rather than as a new black-box algorithm:

- first moment memory for direction;
- second moment memory for per-parameter scale.

This connects directly to Lesson 28's exponential memory framing.

## What Was Confusing

The symbol `v_t` is overloaded. In momentum it means velocity; in Adam it usually means the second moment estimate. Lesson-local symbol tables need to call this out explicitly.

## Notation Issues

- `m_t`, `v_t`, `\hat{m}_t`, and `\hat{v}_t` should stay visible in Adam practice.
- `β_1` and `β_2` need names tied to the memory they control.
- `ε` should be explained as numerical stability, not as another learning-rate-like parameter.

## Concepts To Reinforce

- Exponential moving averages.
- Bias correction from zero initialization.
- Elementwise operations over parameter vectors.
- The difference between optimizer state and model parameters.

## Connections Made

Adam compresses earlier optimizer ideas:

- SGD supplies mini-batch gradient estimates.
- Momentum supplies direction memory.
- RMSProp-style scaling supplies adaptive coordinate-wise step sizes.
