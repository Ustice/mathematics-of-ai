# Papers

This directory is for structured paper-reading notes.

## Template

Use [paper-reading-template.md](paper-reading-template.md) for each paper.

Paper notes should identify:

- Why the paper is worth reading.
- The problem statement.
- The core claim.
- Notation and symbol definitions.
- Mathematical tools used.
- Assumptions.
- Main argument.
- Implementation notes.
- What to reproduce.
- Critique and questions.

## Status Rule

Do not treat a paper as understood because a summary exists. A useful paper note should make the assumptions, notation, and argument inspectable.
