## Goal

Understand curvature as the way the slope changes as you move through parameter space.

The core idea is:

> The gradient tells you which way looks downhill right here. Curvature tells you how quickly that downhill direction is changing.

That matters for optimization because a learning rate that is safe in a flat direction may be too large in a sharply curved direction.

## Motivation

Gradient descent uses first-order information:

```math
θ_{t+1} = θ_t - η∇L(θ_t)
```

The gradient says:

> From here, which direction increases the loss fastest?

So `-∇L(θ_t)` points locally downhill.

But the gradient does not tell us whether the surface is nearly flat, sharply curved, bowl-shaped, ridge-shaped, or saddle-shaped. Two points can have the same gradient magnitude but very different nearby geometry.

That missing information is curvature.

| $∇L(θ)$ | gradient | Vector of first derivatives. |
| $H$ | Hessian | Matrix of second derivatives. |
| $H_{ij}$ | Hessian entry | How the $i$th gradient component changes as $θ_j$ changes. |
| $v$ | direction vector | A direction used to ask about curvature along that direction. |

## One-Dimensional Curvature

Start with a one-dimensional loss:

```math
L(θ) = θ^2
```

The first derivative is:

```math
\frac{dL}{dθ} = 2θ
```

The second derivative is:

```math
\frac{d^2L}{dθ^2} = 2
```

The first derivative gives slope.

The second derivative gives curvature: how quickly the slope changes.

For a quadratic bowl like $θ^2$, curvature is constant. The slope changes at the same rate everywhere.

## Positive, Negative, And Zero Curvature

For a one-dimensional function:

| Second derivative | Shape intuition | Optimization meaning |
| --- | --- | --- |
| $L''(θ) > 0$ | bowl / cup | locally convex; a minimum may be nearby |
| $L''(θ) < 0$ | hill / cap | locally concave; a maximum may be nearby |
| $L''(θ) = 0$ | locally flat or changing slowly | first-order information may dominate |

Examples:

```math
L(θ)=θ^2 \Rightarrow L''(θ)=2
```

positive curvature.

```math
L(θ)=-θ^2 \Rightarrow L''(θ)=-2
```

negative curvature.

```math
L(θ)=3θ+1 \Rightarrow L''(θ)=0
```

zero curvature because the slope is constant.

## Why Curvature Affects Learning Rate

Consider:

```math
L(θ)=aθ^2
```

The gradient is:

```math
\frac{dL}{dθ}=2aθ
```

The curvature is:

```math
\frac{d^2L}{dθ^2}=2a
```

If $a$ is large, the bowl is steep and narrow. A large learning rate can overshoot.

If $a$ is small, the bowl is wide and shallow. A small learning rate can make painfully slow progress.

This is one reason the same learning rate can behave differently across different parameters or different regions of training.

Learning rate schedules from Lesson 30 change step scale over time. Curvature explains why step scale is even a problem: the loss surface may not have the same safe step size everywhere.

## From One Parameter To Many Parameters

Real models have many parameters, so the loss is a function of a vector:

```math
L(θ_1, θ_2, \dots, θ_n)
```

The gradient is a vector:

```math
∇L(θ) =
\begin{bmatrix}
\frac{∂L}{∂θ_1} \\
\frac{∂L}{∂θ_2} \\
\vdots \\
\frac{∂L}{∂θ_n}
\end{bmatrix}
```

Each entry says how the loss changes if you move one parameter while holding the others fixed.

The Hessian asks the next question:

> How do those gradient entries change as the parameters change?

## The Hessian Matrix

The Hessian is the matrix of second partial derivatives:

```math
H =
\begin{bmatrix}
\frac{∂^2L}{∂θ_1^2} & \frac{∂^2L}{∂θ_1∂θ_2} & \cdots & \frac{∂^2L}{∂θ_1∂θ_n} \\
\frac{∂^2L}{∂θ_2∂θ_1} & \frac{∂^2L}{∂θ_2^2} & \cdots & \frac{∂^2L}{∂θ_2∂θ_n} \\
\vdots & \vdots & \ddots & \vdots \\
\frac{∂^2L}{∂θ_n∂θ_1} & \frac{∂^2L}{∂θ_n∂θ_2} & \cdots & \frac{∂^2L}{∂θ_n^2}
\end{bmatrix}
```

The diagonal entries tell you curvature along individual parameter axes.

The off-diagonal entries tell you interaction curvature: how changing one parameter affects the gradient with respect to another parameter.

That is the multivariable version of "the slope is changing."

## A Small Two-Parameter Example

Let:

```math
L(x,y)=3x^2 + y^2
```

The gradient is:

```math
∇L(x,y)=
\begin{bmatrix}
6x \\
2y
\end{bmatrix}
```

The Hessian is:

```math
H=
\begin{bmatrix}
6 & 0 \\
0 & 2
\end{bmatrix}
```

The $x$ direction has larger curvature than the $y$ direction.

Plain English:

> The bowl is narrower along $x$ than along $y$.

A learning rate that is safe for the $y$ direction may be too large for the $x$ direction.

## Interaction Curvature

Now consider:

```math
L(x,y)=x^2 + 2xy + y^2
```

The gradient is:

```math
∇L(x,y)=
\begin{bmatrix}
2x + 2y \\
2x + 2y
\end{bmatrix}
```

The Hessian is:

```math
H=
\begin{bmatrix}
2 & 2 \\
2 & 2
\end{bmatrix}
```

The off-diagonal entries are nonzero. That means the parameters are coupled: changing $x$ changes the gradient with respect to $y$, and changing $y$ changes the gradient with respect to $x$.

This is why high-dimensional optimization is harder than many independent one-dimensional problems glued together.

## Curvature Along A Direction

The Hessian is a matrix, but we often want to ask:

> What is the curvature if I move in this particular direction?

For a direction vector $v$, directional curvature is measured by:

```math
v^T H v
```

You do not need to compute this fluently yet. The important pattern is familiar from quadratic forms:

- $H$ stores the local curvature structure;
- $v$ chooses a direction;
- $v^T H v$ reports the curvature along that direction.

This connects back to covariance geometry: a matrix can encode how a space stretches differently in different directions.

Here the matrix is not a covariance matrix. It is a curvature matrix. But the geometric move is similar.

## Minima, Maxima, And Saddles

At a point where the gradient is zero:

```math
∇L(θ)=0
```

we call the point a critical point.

Curvature helps classify the local shape:

| Hessian behavior | Local shape |
| --- | --- |
| positive curvature in every direction | local minimum candidate |
| negative curvature in every direction | local maximum candidate |
| positive in some directions, negative in others | saddle point |

A saddle point is especially important in machine learning. It can look flat or stationary by the gradient, but it is not a true minimum.

In two dimensions, imagine a horse saddle: curved up in one direction and curved down in another.

## Why We Usually Do Not Build The Full Hessian

For a model with $n$ parameters, the gradient has $n$ entries.

The Hessian has $n^2$ entries.

A tiny model with 1,000 parameters has a Hessian with 1,000,000 entries.

A modern neural network may have millions or billions of parameters. The full Hessian is usually too expensive to store or invert directly.

So in practical ML, Hessian thinking often appears indirectly:

- second-order approximations;
- curvature-aware optimizers;
- diagonal approximations;
- Hessian-vector products;
- learning-rate limits;
- loss-surface diagnostics.

The concept matters even when the full matrix is not explicitly built.

## Connection To Optimizers

SGD follows the noisy downhill direction.

Momentum smooths directions over time.

Adam rescales updates coordinate by coordinate using squared-gradient memory.

Learning-rate schedules change the global step scale over time.

Curvature explains why all of these tricks help: the surface may be steep in some directions, flat in others, noisy in early training, and different later.

No optimizer gets to ignore shape.

## Connection To Programming

A one-dimensional second derivative can be approximated numerically with finite differences:

```ts
function secondDerivative1D(
  loss: (theta: number) => number,
  theta: number,
  h: number,
): number {
  return (loss(theta + h) - 2 * loss(theta) + loss(theta - h)) / (h * h);
}
```

This says:

> Compare the loss slightly to the right, at the point, and slightly to the left. If the middle is much lower than the sides, the curve bends upward.

For real neural networks, we usually use automatic differentiation rather than hand-written finite differences. But finite differences are useful for intuition.

## Summary

First derivative:

```math
\frac{dL}{dθ}
```

tells slope.

Second derivative:

```math
\frac{d^2L}{dθ^2}
```

tells curvature.

Gradient:

```math
∇L(θ)
```

tells the local first-order direction of steepest increase.

Hessian:

```math
H_{ij}=\frac{∂^2L}{∂θ_i∂θ_j}
```

stores local second-order shape information.

Curvature matters because optimization is not just about which direction is downhill. It is also about how safe it is to move a given distance in that direction.
